# Seleccion de imagenes Lote 01

## Objetivo del documento

Dejar cerrado un marco de seleccion rapido para las tres imagenes con mayor impacto comercial inmediato:

- `hero-main`
- `services-hero`
- `contact-hero`

El documento debe servir para elegir una via y una direccion concreta por asset sin improvisar ni reabrir debates de estilo ya resueltos.

## Coherencia de serie

Las tres imagenes deben funcionar como una serie, no como piezas aisladas.

### La serie debe compartir

- la misma familia de materiales: madera, papel, metal discreto, pantalla moderna;
- una luz suave y profesional;
- paleta calida-neutra con apoyo frio muy controlado;
- sensacion de despacho tecnico real;
- composiciones limpias con uno o dos focos claros;
- tecnologia visible, pero nunca exhibicionista.

### La serie no debe mezclar

- una imagen muy editorial con otra demasiado stock;
- un hero muy oscuro con otra imagen luminosa y corporativa;
- una escena hiperrealista con otra claramente artificial;
- un encuadre muy cerrado con otros excesivamente vacios sin razon.

### Regla de serie

Si una imagen individual funciona bien, pero rompe el tono del conjunto, debe descartarse o moverse a otro lote.

## hero-main

### Objetivo visual

Ser la imagen mas fuerte del sitio y dar la primera impresion de despacho tecnico premium con analisis y documentacion visibles.

### Prioridad

Alta.

### Via preferida

Generar.

### Opcion A

Generacion de escena hero de escritorio principal.

Pros:

- permite controlar composicion, materiales y tono;
- facilita una mezcla precisa de monitor, documentacion y mesa;
- reduce riesgo de stock generico.

Contras:

- puede verse artificial si la pantalla o los objetos quedan poco creibles;
- exige seleccion fina entre varias salidas.

### Opcion B

Stock editorial premium de despacho tecnico.

Pros:

- rapidez;
- realismo fotografico inmediato;
- buena opcion si aparece una escena excepcionalmente alineada.

Contras:

- dificil encontrar una pieza con suficiente nivel y sin look corporativo generico;
- riesgo de incoherencia con el resto de la serie.

### Opcion C

Fotografia propia posterior.

Pros:

- maxima singularidad;
- mejor coherencia con la marca a medio plazo.

Contras:

- no resuelve el lote con rapidez;
- requiere localizacion, atrezzo y tiempo.

### Criterio de descarte

Descartar si la imagen:

- parece oficina premium generica y no despacho tecnico;
- muestra demasiados dispositivos;
- usa luz azulada o dramatica;
- hace que la documentacion parezca decorativa en lugar de funcional.

### Checklist minima de validacion

- monitor principal visible y creible;
- documentacion tecnica presente;
- materiales sobrios y de calidad;
- luz lateral suave;
- ninguna estetica hacker o corporativa vacia.

## services-hero

### Objetivo visual

Abrir servicios con una imagen que haga sentir capacidad tecnica, orden y solvencia comercial.

### Prioridad

Alta.

### Via preferida

Generar.

### Opcion A

Generacion de escena de revision tecnica con mesa, monitor y documentos.

Pros:

- permite componer la imagen exactamente para la pagina;
- facilita mostrar servicio y criterio en una sola escena;
- ayuda a evitar stock demasiado abstracto.

Contras:

- riesgo de exagerar la interfaz o volverla poco realista;
- requiere consistencia con `hero-main`.

### Opcion B

Stock premium de escritorio de analisis sobrio.

Pros:

- resolucion rapida;
- posible realismo alto si se encuentra una pieza muy controlada.

Contras:

- muchos resultados de stock para "analisis" se van a ciberseguridad o corporate;
- dificil encontrar documentacion convincente.

### Opcion C

Fotografia propia posterior de mesa de trabajo y documentos.

Pros:

- buena opcion para una segunda iteracion mas exclusiva;
- podria reforzar mejor la autenticidad de servicios.

Contras:

- no es la via mas eficiente para resolver el lote inicial.

### Criterio de descarte

Descartar si la imagen:

- parece dashboard comercial en vez de revision tecnica;
- depende demasiado de pantalla y no muestra contexto documental;
- tiene demasiada oscuridad o saturacion;
- parece ilustracion de ciberseguridad mas que servicio profesional.

### Checklist minima de validacion

- pantalla sobria con lectura tecnica;
- documento o informe visible;
- sensacion de orden y trabajo real;
- composicion tres cuartos o equivalente con buena profundidad;
- tono comercial serio, no decorativo.

## contact-hero

### Objetivo visual

Transmitir confianza, claridad y nivel profesional en el punto previo al contacto.

### Prioridad

Alta.

### Via preferida

Stock.

### Opcion A

Stock editorial de despacho o mesa de reunion ligera.

Pros:

- es la via mas rapida y suficiente para esta pieza;
- permite encontrar escenas luminosas y sobrias con menos complejidad que los heroes tecnicos.

Contras:

- muchos resultados de stock se vuelven demasiado humanos o demasiado corporativos;
- requiere filtro estricto para evitar sonrisas y teatralidad.

### Opcion B

Generacion de escena de contacto sobrio con mesa y documentacion.

Pros:

- permite alinear mejor materiales y color con el resto de la serie;
- util si no aparece stock suficientemente refinado.

Contras:

- no necesita tanta especificidad como para justificarlo de entrada;
- riesgo de perder naturalidad en una imagen que debe sentirse amable y real.

### Opcion C

Fotografia propia posterior de mesa de reunion o escritorio secundario.

Pros:

- buena opcion para una version futura mas exclusiva;
- facil de producir si ya existe un espacio coherente.

Contras:

- no es imprescindible para desbloquear el lote inicial.

### Criterio de descarte

Descartar si la imagen:

- parece recepcion corporativa;
- muestra personas sonriendo o estrechandose la mano;
- tiene demasiado vacio elegante sin informacion util;
- se percibe legalista o institucional.

### Checklist minima de validacion

- mesa o espacio de reunion sobrio;
- documento o carpeta visible;
- dispositivo discreto;
- luz amable pero profesional;
- sensacion de conversacion bien encuadrada.

## Como validar si las tres imagenes funcionan juntas

Revisar las tres en paralelo, no una a una.

### Prueba minima

- colocar las tres miniaturas juntas;
- comprobar si la temperatura de color parece de la misma familia;
- revisar si comparten materiales similares;
- verificar que ninguna domina por dramatismo o por vacio;
- comprobar que las tres parecen pertenecer al mismo estudio o firma.

### Prueba de jerarquia

- `hero-main` debe ser la mas potente;
- `services-hero` debe sentirse igual de seria, pero mas operativa;
- `contact-hero` debe cerrar con mas calma y confianza que con tension tecnica.

## Errores de serie a evitar

- una imagen demasiado luminosa frente a dos demasiado oscuras;
- una imagen muy fotorealista y dos claramente generadas;
- una escena con demasiada gente y otras totalmente objetuales;
- materiales incoherentes: una mesa de marmol frio, otra de madera rustica, otra de oficina generica;
- pantallas con estilos visuales completamente distintos;
- una paleta muy beige y otra muy azul sin punto comun.

## Orden recomendado de ejecucion

### Paso 1

Definir primero la direccion definitiva de `hero-main`.

Motivo:

Es la imagen que fija el nivel de toda la serie.

### Paso 2

Resolver `services-hero` buscando continuidad material y de luz con `hero-main`.

Motivo:

Es la segunda imagen con mas impacto comercial.

### Paso 3

Cerrar `contact-hero` adaptando el tono a una sensacion mas amable, pero sin romper la serie.

Motivo:

Debe acompanar, no competir.

## Decision practica recomendada

Si se quiere avanzar rapido:

- `hero-main`: generar;
- `services-hero`: generar;
- `contact-hero`: stock premium.

Si se quiere maximizar coherencia futura:

- generar primero `hero-main`;
- usar esa imagen como referencia visual para filtrar `services-hero` y `contact-hero`.
