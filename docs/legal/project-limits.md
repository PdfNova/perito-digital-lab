# Project Limits

## Restricciones generales
- No incluir datos personales reales
- No usar nombres de casos reales
- No mencionar afiliaciones institucionales
- No incluir procedimientos internos sensibles
- No prometer resultados imposibles
- No usar lenguaje agresivo o ambiguo

## Restricciones comerciales
- No vender “hackeos”
- No ofrecer accesos a cuentas
- No ofrecer recuperación invasiva de datos de terceros
- No presentar herramientas sensibles como reclamo principal

## Restricciones editoriales
- Todo contenido debe poder anonimizarse
- Todo ejemplo debe ser genérico o ficticio
- Toda explicación debe priorizar utilidad legal/técnica
