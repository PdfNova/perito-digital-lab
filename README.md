# perito-digital-lab

Base documental para una marca digital discreta y profesional centrada en investigacion digital, analisis forense, OSINT, trazabilidad tecnologica y documentacion tecnica/pericial.

## Objetivo del proyecto

Definir una presencia digital sobria y util que transmita autoridad tecnica sin recurrir a lenguaje agresivo, promesas ambiguas ni reclamos de herramientas sensibles.

Esta fase no desarrolla producto ni interfaz. Su objetivo es dejar clara la direccion estrategica y editorial antes de intervenir la carpeta `app/`.

## Alcance de esta fase

- Fijar vision, propuesta de valor y limites del proyecto.
- Establecer pilares de contenido reutilizables para web, blog y videos.
- Unificar tono, restricciones y prioridades de trabajo.
- Preparar una base documental que permita construir la web con coherencia.

## Principios del proyecto

- Discrecion antes que exposicion.
- Autoridad tecnica antes que marketing.
- Utilidad documental antes que volumen de contenido.
- Claridad expositiva antes que jerga innecesaria.
- Anonimizacion y prudencia antes que detalle sensible.

## Estructura actual del repositorio

- `app/`: espacio reservado para la futura web. No se modifica en esta fase.
- `assets/`: recursos visuales y piezas graficas de apoyo.
- `content/blog/`: borradores, articulos y lineas editoriales para contenidos escritos.
- `content/videos/`: guiones, estructura narrativa, voz y subtitulos para piezas audiovisuales.
- `docs/branding/`: definiciones de posicionamiento, vision y propuesta de valor.
- `docs/legal/`: limites de contenido, alcance y criterios de prudencia documental.
- `docs/research/`: pilares tematicos y mapa base de contenido.
- `scripts/`: automatizaciones futuras para soporte editorial o de publicacion.

## Documentos estrategicos incluidos

- `docs/branding/project-vision.md`: marco general de vision, publico, posicionamiento y criterios de exito.
- `docs/branding/value-proposition.md`: propuesta de valor, diferenciales, mensajes clave y limites comerciales.
- `docs/research/content-pillars.md`: pilares editoriales para web, blog y videos.
- `docs/legal/project-limits.md`: restricciones de contenido, privacidad, prudencia legal y exclusiones.
- `AGENTS.md`: instrucciones de trabajo para mantener consistencia al ampliar el proyecto.

## Criterios editoriales base

- Escribir en espanol tecnico, sobrio, claro y directo.
- Evitar enfoque de hacking, espectacularizacion o promesas de acceso.
- No usar nombres reales, datos personales ni referencias institucionales.
- Priorizar explicaciones utiles para casos, evidencia, trazabilidad y documentacion.
- Reutilizar conocimiento anonimizado en formatos adaptables a web, blog y video.

## Estado actual

El repositorio queda preparado como base documental inicial. La siguiente fase deberia traducir estos documentos en arquitectura web, mensajes de portada, estructura de servicios y sistema editorial sin alterar los limites aqui definidos.
